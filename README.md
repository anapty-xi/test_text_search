# FastAPI Text Search App

Полностью отвечает поставленному заданию, асинхронное апи с clean architecture, автодоки Fastapi, функциональные тесты. Пострался полностью отразить свой подход к разработке, поэтому многое можно назвать оверинжиниренгом.

---

## Технологический стек

* **API**: FastAPI, SQLAlchemy (Async), Dependency Injector
* **Databases**: PostgreSQL, Elasticsearch
* **Environment**: Docker, Docker Compose
* **QA / Linting**: Pytest, Httpx, Mypy

---

## Инструкция по запуску

Проект полностью контейнеризирован. Для сборки и старта всех сервисов выполните следующие шаги:

### 1. Настройка окружения
Убедитесь, что в корне проекта создан файл `.env` с необходимыми переменными (пример структуры):
```env
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
POSTGRES_URL=postgresql+asyncpg://postgres:postgres@postgres:5432/test_text_search

ELASTIC_CLIENT=http://elasticsearch:9200
ELASTIC_INDEX_NAME=posts
```

### 2. Сборка и запуск контейнеров
Запустите проект одной командой (флаг `-d` запустит контейнеры в фоновом режиме):
```bash
docker compose up --build -d
```
*При старте приложение автоматически проверит состояние баз, применит миграции Alembic, создаст поисковый индекс в Elasticsearch с поддержкой русского языка и наполнит БД начальными данными из CSV-файла.*

## Запуск тестов и линтеров

Функциональные тесты написаны с использованием `pytest`. Они запускаются внутри окружения работающего контейнера.

### 1. Прогон тестов (Pytest)
Убедитесь, что контейнеры запущены, и выполните команду для старта тестов:
```bash
docker compose exec -ti api pytest .
```

### 2. Проверка типов (Mypy)
Для запуска статического анализатора типов в строгом (`strict`) режиме выполните:
```bash
docker compose exec -ti api mypy .
```
